MundeApp is a native Android application built for travel agency employees and managers. It gives staff a mobile interface to maintain a live catalog of travel destinations: adding new listings, editing existing ones, and removing outdated entries — all backed by Firebase.

## Who it's for

MundeApp is designed for **travel agency staff** who need to manage a destination catalog on the go. Whether you're adding a new beach resort or updating a price, the app keeps every logged-in device in sync through Cloud Firestore.

## What problems it solves

- **Centralized catalog** — All destinations live in a single Firestore collection, accessible to every authenticated team member
- **Image hosting** — Hero photos are uploaded to Firebase Cloud Storage and linked to each destination record
- **Real-time sync** — The destination list refreshes automatically whenever you return to the main screen
- **Secure access** — Firebase Authentication (email/password) ensures only authorized staff can create or modify listings

## App screens

MundeApp has four screens that cover the complete user journey from sign-in to destination management.

| Screen | Class | Description |
|---|---|---|
| **Login** | `LoginActivity` | The app launcher screen. Enter your email and password to sign in, or tap the register button to create a new account. |
| **Register** | `RegisterActivity` | Create a new staff account using an email address and password. On success, the screen closes and returns you to Login. |
| **Destinations list** | `MainActivity` | The main screen. Shows all destinations in a scrollable list with options to edit or delete each entry. A floating action button opens the form to add a new destination. The toolbar provides a logout option. |
| **Add / Edit destination** | `FormDestinationActivity` | A form to create a new destination or edit an existing one. Fields include name, country (spinner), price, description, and a hero image picked from the device gallery. |
