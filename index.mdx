## What MundeApp does

MundeApp gives travel agency staff a mobile interface to maintain a live catalog of destinations. Each destination has a name, country, price, description, and a hero photo. Staff members log in with their agency credentials, then add, edit, or remove listings — changes are reflected in real time across all logged-in devices.

<Steps>
  <Step title="Create a Firebase project">
    Set up Firebase Authentication, Firestore, and Cloud Storage in the [Firebase Console](https://console.firebase.google.com). See the [Firebase setup guide](/firebase/setup) for step-by-step instructions.
  </Step>
  <Step title="Clone and configure the app">
    Clone the repository, add your `google-services.json` file, and open the project in Android Studio. See the [quickstart](/quickstart) for details.
  </Step>
  <Step title="Build and run">
    Run the app on a device or emulator (Android 11+). The login screen appears first — register a new account to get started.
  </Step>
  <Step title="Add your first destination">
    Tap the **+** button on the main screen, fill in the destination details, pick a photo from your gallery, and tap **Save**. Your destination is now live in Firestore.
  </Step>
</Steps>

## Key features

- **Email/password authentication** — Secure sign-in and registration via Firebase Auth
- **Destination catalog** — Full CRUD for travel listings stored in Cloud Firestore
- **Photo uploads** — Attach a hero image to every destination via Firebase Cloud Storage
- **Real-time sync** — Destination list refreshes automatically whenever you navigate back to the main screen
- **Material Design UI** — Clean, responsive interface with custom purple/pink branding
- **Auth-guarded screens** — Users who are not logged in are automatically redirected to the login screen
