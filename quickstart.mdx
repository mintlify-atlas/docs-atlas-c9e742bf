---
title: "Quickstart"
description: "Get MundeApp running on a device or emulator in a few steps — clone the repo, connect Firebase, and make your first build."
---

This guide walks you through setting up the MundeApp Android project from scratch and running it on a physical device or emulator.

<Info>
  **Prerequisites**

  - Android Studio Hedgehog (2023.1.1) or later
  - JDK 11
  - Android device or emulator running API 30 (Android 11) or higher
  - A Firebase account at [console.firebase.google.com](https://console.firebase.google.com)
</Info>

<Steps>
  <Step title="Clone the repository">
    Open a terminal and clone the project:

    ```bash
    git clone https://github.com/DiegoMajano/TravelAgencyApp.git
    ```
  </Step>

  <Step title="Create a Firebase project">
    Before you can build the app, you need a Firebase project with Authentication, Firestore, and Cloud Storage enabled.

    See the [Firebase setup guide](/firebase/setup) for step-by-step instructions on creating the project and enabling the required services.
  </Step>

  <Step title="Add google-services.json">
    After setting up Firebase, download the `google-services.json` configuration file from the Firebase Console and place it in the `app/` directory:

    ```
    TravelAgencyApp/
    └── app/
        └── google-services.json   ← place it here
    ```

    <Note>
      `google-services.json` is not included in the repository. It contains project-specific API keys and identifiers that should not be committed to version control.
    </Note>
  </Step>

  <Step title="Open the project in Android Studio">
    Launch Android Studio and select **File → Open**. Navigate to the root `TravelAgencyApp/` folder (the one containing `settings.gradle.kts`) and open it.

    Do not open the `app/` subdirectory — Android Studio needs the root to resolve the Gradle project structure.
  </Step>

  <Step title="Build the project">
    Android Studio will trigger a Gradle sync automatically. This downloads all dependencies declared in `app/build.gradle.kts`, including:

    | Dependency | Version |
    |---|---|
    | `firebase-auth` | 24.0.1 |
    | `firebase-firestore` | 26.1.2 |
    | `firebase-storage` | 22.0.1 |
    | `material` | 1.11.0 |
    | `recyclerview` | 1.3.2 |
    | `glide` | 4.16.0 |

    Wait for the sync to complete before proceeding. You can monitor progress in the **Build** tool window at the bottom of the IDE.
  </Step>

  <Step title="Run on a device or emulator">
    Connect a physical Android device via USB (with USB debugging enabled), or start an Android Virtual Device (AVD) from **Device Manager**.

    Select your device from the run target dropdown and click **Run** (or press `Shift+F10`).

    <Tip>
      If you don't have a physical device, use the Android Emulator with an AVD configured to API 30 or higher. In the AVD Manager, create a new virtual device and choose a system image with API level 30+.
    </Tip>
  </Step>

  <Step title="Register an account">
    The app opens on the **Login** screen. Tap **Register** to create a new staff account with an email address and password. After a successful registration, you are returned to the Login screen.

    Sign in with your new credentials to access the destinations list.
  </Step>

  <Step title="Add your first destination">
    Once logged in, you'll see the main destinations screen. Tap the **+** floating action button in the bottom-right corner to open the destination form.

    Fill in the name, country, price, and a description of at least 20 characters, then pick a hero image from your device gallery. Tap **Save** — the destination is uploaded to Firestore and its image stored in Firebase Cloud Storage.
  </Step>
</Steps>
